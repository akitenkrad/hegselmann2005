## socsim フレームワーク上のモデル

シミュレーションエンジンは社会シミュレーション基盤 [rs-social-simulation-tools](https://github.com/akitenkrad/rs-social-simulation-tools) (socsim) 上に構築する (git 依存，commit は `Cargo.lock` で固定)．正準の Hegselmann–Krause モデルは**完全グラフ・非空間**モデルであるため，`socsim-core` (traits) と `socsim-engine` (Simulation / Builder) のみを使い，**`socsim-grid` も `socsim-net` も使わない**．

使用する socsim API:

- `WorldState` — `OpinionWorld` が `agent_ids` / `clock` / `clock_mut` を実装する．意見ベクトル `opinions: Vec<f64>`，信頼幅 `eps`，`MeanOperator`，`last_max_delta` を保持する．
- `Mechanism` / `Phase::Interaction` — `BoundedConfidenceUpdate` が `Interaction` フェーズで発火する．
- `SequentialScheduler` — `AgentId` 昇順で活性化する．更新は同期的なので活性化順序は結果に無関係であり，決定論的な逐次スケジューラを使い `ctx.agent_order` は無視する．
- `StepContext::request_stop` / `Simulation::stop_requested` — 収束時の早期停止．
- `StepContext::scratch` / `Simulation::run_observed` — ステップ結果 (`max_delta`, `converged`) をドライバへ受け渡す．
- `SimRng` / `derive_seed` — `derive_seed(root, &[0])` を初期意見の RNG に，`derive_seed(root, &[1])` を engine RNG (ランダム平均 R のみ使用) に充てる．

## 有界信頼更新 (同期)

一般化更新規則 (論文 式(2)):

```
x_i(t+1) = M( { x_j(t) : j ∈ I(i, x(t)) } ),   I(i, x) = { j : |x_i(t) − x_j(t)| ≤ ε }
```

ここで `M` は選択された平均演算子．メカニズムは**同期 (simultaneous) 更新**を実装する: `prev = world.opinions.clone()` をスナップショットし，各エージェントの信頼集合と新意見を `prev` から計算したのち，全新意見を一括代入する．同期更新は「m 個のグループが順に更新する手続きの極限ケース」(論文 §4) であり，結果は活性化順序に依存しない．ゆえに決定論的な `SequentialScheduler` で十分で，スケジューラは結果に影響しない．完全グラフ走査は 1 ステップ `O(n²)`．論文の `n = 625` でも軽量である．

## 平均演算子

信頼集合内の非空多重集合 `S` (`m = |S|`，常に自分自身を含む) に対し:

| 演算子 | 定義 | 備考 |
|---|---|---|
| A (算術) | `(1/m) Σ s` | `= P_1`，2002 年 HK 規則 |
| G (幾何) | `(Π s)^{1/m}` | `= P_0`，`s > 0` を要求 (対数空間で計算) |
| H (調和) | `m / Σ (1/s)` | `= P_{-1}`，`s > 0` を要求 |
| P_p (べき / Hölder) | `((1/m) Σ s^p)^{1/p}` | `p ≠ 0` |
| R (ランダム) | `Uniform(min S, max S)` | RNG を使う唯一の演算子 |

系統的不等式: `p ≥ 1` で `P_{-∞}(min) ≤ H ≤ G ≤ A ≤ P_p ≤ P_{∞}(max)`．

A/G/H/P は決定論的で，`max|Δx| < tol` で停止を要求する．R は毎ステップ非決定的 (RNG を引く) なので停止を要求せず `max-iterations` まで回す (論文 Observation 6, Fact 5)．G/H を well-defined に保つため，これらの初期意見は開区間 `]1e-9, 1[` から引き，全値が正であることを保証する．

## 評価指標

- `n_occupied_classes` — 意見を `1e-4` 刻みのビンに分割し非空ビン数を数える (生存した相異なる意見数; 論文 Fact 1/3, Observation 1, Fig. 4–7)．
- `phase` — consensus (1) / polarization (2) / plurality (≥3)．占有クラス数から分類する．
- `mean_opinion`, `variance` — 安定後プロファイルの要約統計量．
- `consensus_brink` — 占有クラス数が初めて 1 になる最小 ε (ε* の数値推定; 論文 Observation 1, Fact 4)．sweep サマリで計算する．

## 再現性・決定論性

同一シードなら実行は決定論的 (同じ意見軌跡)．A/G/H/P はステップ内で RNG を使わず完全に決定論的，R も engine RNG ストリームがシードで固定されるためシードごとに再現する．

## 論文再現

`reproduce` サブコマンドは論文の主要実験を横断して Rust バイナリを連結実行し，観測値と期待値の演算子ごとの相を突き合わせて図を描く ([可視化](visualization.ja.md) ドキュメント参照)．設計上は，ネットワーク版 (`socsim-net`，ER/WS/BA) で信頼集合を `ネットワーク近傍 ∩ 意見距離` に変更する余地と，PAM levelling の解析的検証という拡張点も残している．
