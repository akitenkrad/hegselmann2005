# Opinion Dynamics Driven by Various Ways of Averaging — Hegselmann & Krause (2005)

A reimplementation of the continuous opinion-dynamics model of Hegselmann & Krause (2005), "Opinion Dynamics Driven by Various Ways of Averaging" (*Computational Economics* 25, 381–405). Each agent holds an opinion `x_i ∈ [0,1]` and, every step, aggregates the opinions in its confidence set `I(i) = { j : |x_i − x_j| ≤ ε }` using one of five averaging operators — arithmetic (A), geometric (G), harmonic (H), power/Hölder (P_p), or random (R). The model lives on the complete graph (everyone-sees-everyone, non-spatial); the confidence set is recomputed every step from opinion distance. The simulation is written in Rust on top of the [socsim](https://github.com/akitenkrad/rs-social-simulation-tools) framework, and the visualization tools in Python.

## Install & Quick start

```bash
# Build the Rust simulation
cargo build --release

# Run with default settings (n=625, ε=0.15, arithmetic mean, seed=42)
cargo run --release -- run --n 625 --eps 0.15 --mean A --seed 42

# Install the Python visualization tools (at the workspace root)
uv sync

# Visualize the most recent run (opinion trajectory + metrics)
uv run hegselmann-tools visualize
```

## Scope

This repository implements the core bounded-confidence model with A/G/H/P/R averaging on the complete graph (the `run` subcommand and the three-phase transition), the `sweep` over ε with the Python `visualize` / `visualize-sweep` tools, and a one-shot `reproduce` that runs the paper's headline experiments and renders the figures with an observed-vs-expected per-operator regime check. A network variant on `socsim-net` and analytic PAM-levelling checks are possible extensions; clean extension points are kept throughout.
